//Simple call
AutoComplete();

//Default args
{	"selector": ["input[data-autocomplete]"],
	"method": "GET",
	"paramName": "q",
	"type": "JSON",
}

//Call with custom args
AutoComplete({
	"paramName": "query",
	"method": "POST",
	"type": "HTML"
});
//Simple declaration
<input type="text" data-autocomplete="url" / >

//Specific attributes
<input type="text" data-autocomplete="url"
	data-autocomplete-method="POST"
	data-autocomplete-type="HTML"
	data-autocomplete-param-name="query2"/ >

  • data-autocomplete (url)			is required
  • data-autocomplete-method (GET|POST)	is optional
  • data-autocomplete-type (JSON|HTML)	is optional
  • data-autocomplete-param-name		is optional

selector

It's an array. You can give many selector like with jQuery (#id, .class, tag, [attr], â€¦) The default value is input[data-autocomplete].

method

Accept tow values, GET or POST only. The default value is GET.

paramName

paramName is the argument name send when the ajax executed. The default value is q.

type

type accept two values. JSON or HTML.

  • JSON

    The AJAX response is parsed with JSON.parse(). Two possibilities :

    • It's a JSON array. li tag are pushed in ul tag and input autocompleted put the li.value clicked.
      Example:
      ["AARON","ABDUL","ABE","ABEL","ABRAHAM","ABRAM","ADALBERTO","ADAM","ADAN","ADOLFO","ADOLPH"]
    • It's a JSON object. li tag are pushed in ul tag and input autocompleted put the li[data-value] clicked. It's the index of oblect.
      Example:
      {"bar@fake.io":"Bar","baptiste@fake.io":"Baptiste","zang@fake.io":"Zang"}

  • HTML

    No verification on the response. The content is setted into div. The response format give be an ul tag with simple li children.


    Example:
    <ul>
    	<li><a href='http://github.com' target='_blank'>AARON</a></li>
    	<li><a href='http://github.com' target='_blank'>ABDUL</a></li>
